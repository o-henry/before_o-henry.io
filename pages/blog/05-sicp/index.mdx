# Building Abstractions with Procedures

제대로 된 컴퓨터 시스템은 모듈방식(modular manner)에 따라 부품 단위로 만들고 고쳐 쓰고 갈아 끼울 수 있도록 설계되어 있다.

<hr />

프로그램을 짤 때 우리는 프로시저와 데이터라는 두가지를 쓴다. 데이터는 프로시저에서 쓰려는 물건이고, **프로시저**란 데이터를 처리하는 규칙을 적은 것이다.

<hr />

- **primitive expression** ⇢ 가장 단순한 것을 나타낸다
- **means of combination** ⇢ 간단한 것을 모아 복잡한 것으로 만든다
- **means of abstraction** ⇢ 복잡한 것에 이름을 붙여 하나로 다룰 수 있게 간추린다.

<hr />

```scheme
(+ 137 349) ;137 + 349 = 486
   └── 연산자(operator)
       └── 피연산자(operand)  ⇢ 엮은식(combination)
```

위와 같은 방식을 **prefix notation** 라고 한다.

```scheme
(define pi 3.141592)

(define radius 10)

(* pi (* radius radius)) ;314.159
```

`define` 프로시저로 변수를 저장하며, 생성한 이름으로 데이터에 접근 가능하다. 어떤 값에 이름을 붙여두었다가 뒤에 이름으로 필요한 값을 찾아 쓸 수 있다는 말은 실행기 속 어딘가에
name-object 를 저장해둔 메모리가 있다는 뜻이다. 이런 기억공간을 **environment** 라고 한다.
